Web运作原理探析

    1. 编辑HTML文档
    2. 解析并运行JSP代码
    3. 解析并运行JavaScript代码
    4. 发送HTTP请求，并接受HTTP响应
    5. 解析并展示HTML文档
    6. 编译Java源程序代码

  • 以下哪些属于Web服务器的功能？（多选）1、3

    1. 接受HTTP请求，并发送HTTP响应
    2. 编译Java源程序代码
    3. 动态加载并执行程序代码
    4. 运行网页中的JavaScript脚本
    5. 展示网页中的图片
    6. 解析HTML文档

  • 关于HTTP协议，以下哪些说法正确？（多选）2、3、6、7、8

    1. HTTP响应的正文部分必须为HTML文档
    2. HTTP响应的正文部分可以是任意格式的数据，如HTML、JPG、ZIP、MP3、XML和EXE数据等
    3. HTTP协议规定服务器端默认情况下监听TCP80端口
    4. HTTP协议规定了HTML的语法
    5. HTTP协议是由Microsoft公司制定的
    6. 浏览器与Web服务器之间的通信遵循HTTP协议
    7. HTTP是"Hypertext Transfer Protocol"的缩写
    8. HTTP请求的正文部分可以是任意格式的数据，如HTML、JPG、ZIP、MP3、XML和EXE数据等

  • HTML与浏览器是什么关系？（单选）2

    1. 浏览器是HTML文档的编辑器，可以用浏览器来编写HTML页面
    2. 浏览器是HTML的解析器，能够解析HTML文件，并在窗口中展示网页
    3. 浏览器是HTML的编译器和运行器，能够把HTML文件编译成可执行文件，然后执行它

  • 以下哪些属于Web服务器端编程技术？（多选）1、3、6、7

  • 在Web服务器端有一个index.html文件，该文件包含超级链接：<a herf="hello.html">hello.html(纯文本网页)</a>，当浏览器请求访问index.html文件时，以下哪些说法是正确的？（多选）2、4

    1. 浏览器收到来自服务器的index.html文件后，对其解析，根据以上<a>标记中的href属性值，再自动发出一个要求访问hello.html文件的HTTP请求
    2. 当浏览器展示index.html页面时，如果用户选择网页上的"hello.html(纯文本网页)"超级链接，浏览器会发出一个要求访问hello.html文件的HTTP请求。
    3. 按照HTTP/1.1通信时，浏览器每次向Web服务器发送HTTP请求时，都必须先建立FTP连接
    4. 按照HTTP/1.1通信时，浏览器与Web服务器建立了FTP连接后，浏览器可以利用该连接发送多个HTTP请求

  • 有一种协议规定：如果客户端发送一行字符串“date”，服务器端就返回当前日期信息，如果客户端发送一行字符串“exit”，服务器端就结束与客户端的通信。这种协议应该是哪一层的协议？（单选）2

    1. 网络互联层
    2. 应用层
    3. 传输层

  • 关于Web服务，以下哪些说法正确？（多选）3、5、7

    1. 浏览器是访问Web服务的主要客户程序
    2. 为了使SOAP请求能变为HTTP请求的正文部分，SOAP请求必须用HTML语言编写
    3. SOAP请求和SOAP响应都用XML语言来编写
    4. Web服务器负责创建符合SOAP协议的SOAP响应
    5. Web服务可以借助Web服务器来发布
    6. 从一个抽象层面来理解，所有Web应用都可以看作是Web服务
    7. Web服务架构是一种支持异构软件系统之间通信的分布式架构

  • 关于HTTP请求参数，以下哪些说法正确？（多选）3、5

    1. 在GET请求方式下，HTTP请求参数位于HTTP请求的正文部分
    2. 在一个HTTP请求中，只能包含一个HTTP请求参数
    3. 在POST请求方式下，HTTP请求参数位于HTTP请求的正文部分
    4. HTTP请求参数用来指定浏览器请求访问的HTML文件
    5. HTTP请求参数为客户端向Web服务器传递数据提供了便利

  • Web服务与Web应用有哪些区别？（多选）1、2、5

    1. Web服务基于SOAP协议，Web应用基于HTTP协议
    2. Web服务的客户端可以是任意的软件系统，而Web应用的客户端主要是浏览器
    3. Web服务的概念比Web应用的概念出现得更早
    4. Web服务和Web应用必须分别通过不同的服务器来发布
    5. Web应用返回的响应结果主要是HTML文档，而Web服务返回的响应结果实际上是方法调用的返回值

Tomcat简介

  • 关于Servlet规范，以下哪些说法是正确的？（多选）2、3、5、6、7

    1. Servlet规范属于HTTP协议的一部分
    2. Servlet规范由Oracle公司制定
    3. Servlet规范制定了标准Servlet接口
    4. Servlet规范规定Servlet容器的配置文件为conf子目录下的server.xml文件
    5. Tomcat是符合Servlet规范的优秀Servlet容器
    6. Tomcat的组成结构（包括Server、Service、Engine和Connector等组件）是由自身的实现决定的，与Servlet规范无关
    7. Servlet规范规定Servlet实例运行在Servlet容器中

  • 以下哪些选项属于Servlet容器的功能？（多选）1、2、3、4、5、6

    1. 接收并解析客户要求访问特定Servlet的请求
    2. 创建一个包含客户请求信息的ServletRequest对象
    3. 创建一个ServletResponse对象，Servlet用它来生成响应结果
    4. 创建Servlet对象
    5. 调用Servlet对象的service（）服务方法
    6. 把Servlet对象生成的响应结果发送给客户

  • 关于Servlet，以下哪些说法是正确的？（多选）1、4

    1. Servlet是运行在服务器端的小插件
    2. 当客户请求访问某个Servlet时，服务器就把特定Servlet类的源代码发送给浏览器，由浏览器来编译并运行Servlet
    3. Tomcat是目前惟一能运行Servlet的服务器。其他服务器如果希望运行Servlet，就必须与Tomcat集成
    4. Servlet规范规定，标准Servlet接口有个service（）方法，它有两个参数，分别为ServletRequest和ServletResponse类型

  • 当Tomcat作为独立Servlet容器运行时，有哪些特点？（多选）1、2

    1. Tomcat在一个Java虚拟机进程中独立运行
    2. Tomcat是一个独立的Web服务器，直接与客户端通信，负责接收客户请求和发送响应结果
    3. Tomcat与Servlet分别运行在不同的Java虚拟机进程中
    4. 无须启动任何Java虚拟机进程，就能直接运行Tomcat服务器程序

  • 当Tomcat作为其他Web服务器的进程内或者进程外的Servlet容器来运行时，有哪些共同特点？（多选）1、4

    1. 其他Web服务器直接与客户端通信。如果客户端请求访问Servlet，Tomcat的Web服务器插件就把请求转发给Servlet容器组件
    2. Tomcat直接与客户端通信。如果客户端请求访问的不是Servlet，Tomcat的Web服务器插件就把请求转发给其他Web服务器
    3. Tomcat总是在一个Java虚拟机进程中独立运行
    4. Tomcat本身不是一个独立的Web服务器，可以为其他不支持Java的Web服务器提供运行Java Servlet的功能

  • 为什么安装Tomcat时要先安装JDK？（多选）1、2

    1. Tomcat作为Java程序，它的运行离不开JDK提供的Java虚拟机
    2. Tomcat6.x以下的版本在运行时利用JDK提供的Java编译器来动态编译JSP代码
    3. Tomcat利用JDK来接收HTTP请求
    4. Tomcat利用JDK来发送HTTP响应结果

  • 一个用户安装了Tomcat，但无法启动Tomcat，可能是由于哪些原因引起的？（多选）1、2、3、4、6

    1. 没有安装JDK
    2. Tomcat与JDK的版本不匹配，例如Tomcat9.x要求使用JDK8或者以上版本
    3. 没有设置JAVA_HOME系统环境变量
    4. 没有设置CATALINA_HOME系统环境变量
    5. 没有安装浏览器
    6. Tomcat的配置文件server.xml中的配置内容有错误

JSP技术

  • 关于JSP，以下哪些说法正确？（多选）2、3

    1. 当客户端请求访问一个JSP文件，Servlet容器会读取文件系统中的JSP文件，然后把它的源代码作为响应正文发送给客户
    2. Servlet容器必须先把JSP文件编译为Servlet，然后才能运行它
    3. 用JSP来动态生成网页，比用Servlet来动态生成网页更加方便
    4. JSP中只能包含标记，不能包含Java程序代码

  • 一个JSP文件需要引入java.io.File类和java.util.Date类，以下哪些选项的语法是正确的？（多选）1、3

    1. <%@ page import="java.io.File, java.util.Date" %>
    2. import java.io.File；import java.util.Date；
    3. <%@ page import="java.io.File" %>；<%@ page import="java.util.Date" %>
    4. <%@ page import="java.io.File; java.util.Date" %>

  • 关于静态包含和动态包含，以下哪些说法正确？（多选）1、3、5

    1. 静态包含的语法为<%@ include file="目标组件的URL" %>
    2. 静态包含的目标组件可以为JSP文件、HTML文件和Servlet
    3. 对于静态包含，Servlet容器先把目标组件的源代码融合到JSP源组件中，然后对JSP源组件进行编译
    4. 对于动态包含，Servlet容器先把目标组件的源代码融合到JSP源组件中，然后对JSP源组件进行编译
    5. 对于动态包含，Servlet容器会分别编译和运行JSP源组件和JSP目标组件。JSP目标组件生成的响应结果被包含到JSP源组件的响应结果中

  • 关于会话的销毁，以下哪些说法正确？（多选）1、3

    1. 如果服务器端执行了HttpSession对象的invalidate（）方法，那么这个会话被销毁
    2. 当客户端关闭浏览器进程，服务器端会探测到客户端关闭浏览器进程的行为，从而立即销毁相应的HttpSession对象

    2. Java Servlet API制定了负责对会话进行持久化的接口
    3. 对会话进行持久化，可以及时释放处于不活动状态的HttpSession对象占用的内存
    4. 对会话进行持久化，可以使得Web应用重启后，客户端仍然能继续重启前的会话
